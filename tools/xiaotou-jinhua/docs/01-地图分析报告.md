## 3. 关键发现：这是平台托管脚本型地图

### 3.1 `war3map.j` 仅为桩脚本

- 解压后只有 **116 字节**，正常 RPG 地图的主脚本通常为 **数百 KB ~ 数 MB**。
- 使用 IMPLODE 压缩且当前环境未能还原出可读 JASS 源码（121 字节压缩体）。
- 同目录 3 个 `.w3x` 版本的 `war3map.j` 大小**完全一致**，说明这是统一的平台打包方式，而非本地完整脚本地图。

**结论：角色属性、进化数值、经济系统等核心逻辑，大概率不在 `.w3x` 文件内，而在 KK/网易对战平台的云脚本或运行时加载模块中。**

### 3.2 物编数据极少

- 自定义单位/技能修改量极小，**不可能**承载完整 RPG 数值体系。
- 大量数值（英雄属性、进化等级、存档加成等）只能在运行时脚本或平台存档中维护。

### 3.3 `war3mapMisc.txt` 中有价值的全局常量

已提取的关键配置（节选）：

```
MaxHeroLevel=88888888      # 英雄等级上限极高
GrantHeroXP=0              # 关闭原生经验获取
GrantNormalXP=0
HeroFactorXP=0
StrAttackBonus=0           # 关闭原生三维战斗加成
AgiAttackSpeedBonus=0
IntManaBonus=0
DropItemRange=9999
PickupItemRange=9999
GiveItemRange=9999
MaxUnitSpeed=522
```

说明：地图作者已在物编层关闭大量魔兽原生成长机制，**实际属性完全由自定义脚本驱动**。

### 3.4 `main.lua` 内容

仅为 World Editor 调试用的 **JASS 句柄泄漏检测器**（按 End 键输出句柄统计），与改属性无关。

## 5. 对「修改角色属性和数值」的影响

| 方式 | 可行性 | 说明 |
|------|--------|------|
| **直接改 `.w3x` 物编** | ⛔ 极低 | 几乎没有属性定义；改了 `Misc.txt` 常量也影响有限 |
| **解包改 JASS 再打包** | ⛔ 当前不可行 | `war3map.j` 不可读且仅为桩；真实脚本不在图内 |
| **World Editor 打开编辑** | ⚠️ 有限 | 可改地形/装饰/少量单位/Misc 常量，**改不了进化/存档/核心数值** |
| **运行时内存修改** | ⚠️ 可能 | 需逆向游戏内存结构；平台版可能有校验 |
| **平台存档修改** | ❓ 待确认 | 若属性存于 KK 服务器存档，需分析存档格式 |
| **外挂/注入式辅助** | ⚠️ 可能 | 技术可行但涉及平台 TOS；联机风险高 |
| **基于「修改版」二次打包** | ❓ 待确认 | 需你说明「修改版」已改了什么 |
