## 1. 目标定义（待你确认）

「修改游戏里面角色的属性和数值」可能指以下不同场景，**技术路线完全不同**：

| 场景 | 示例 | 典型技术 |
|------|------|----------|
| A. 局内实时改属性 | 游戏中把攻击力改成 99999 | 内存读写 / 脚本注入 |
| B. 改地图默认数值 | 让所有玩家初始攻击 +50% | 改 `.w3x` 物编或 Misc 常量 |
| C. 改平台存档 | 永久天赋、进化等级、金币存档 | 存档文件解析 + 加密处理 |
| D. 单机练习/私服 | 本地不走上平台校验 | 改图 + 本地启动器 |
| E. 做「修改版」地图给别人玩 | 发布新 `.w3x` | 地图编辑 + 脚本逆向 |

**请你先确认属于哪一种或哪几种组合。**

## 2. 方案对比

### 方案一：地图静态修改工具（改 `.w3x`）

**原理**：解包 MPQ → 修改物编/常量 → 重新打包 → 替换地图文件。

**本图适用性：⛔ 很低**

| 优点 | 缺点 |
|------|------|
| 不依赖运行时，改完即生效 | 本图几乎没有属性定义 |
| 技术路径清晰 | 真实逻辑在云脚本，改图无效 |
| 可版本管理 diff | 重新打包后平台可能校验哈希 |

**实际能改的内容（已验证）**：
- `war3mapMisc.txt`：如 `MaxHeroLevel`、拾取范围、经验开关
- 少量 `war3map.w3u` / `war3map.w3a` 条目
- 地形、装饰、UI 字体

**不能改的内容**：
- 进化路线与数值
- 英雄成长曲线
- 存档/商城加成
- 波次怪物属性

**工作量估计**：1–2 周（含 MPQ 打包工具链），但**收益极低**。

---

### 方案二：运行时内存辅助（类似 Cheat Engine）

**原理**：游戏运行时扫描/锁定内存地址，修改单位属性结构体。

**本图适用性：⚠️ 中等（需大量逆向）**

| 优点 | 缺点 |
|------|------|
| 可改局内实时属性 | 需找稳定内存地址/指针链 |
| 不依赖解包脚本 | 游戏更新后地址失效 |
| 对云脚本地图仍可能有效 | KK 平台可能有反作弊 |

**技术步骤**：
1. 确认运行环境（平台客户端 vs 纯本地 War3）
2. 用 Cheat Engine 附加进程，搜索已知数值（当前攻击力、金币等）
3. 找到单位属性基址或 Lua/JASS 层 userdata
4. 封装为 GUI 工具（读写 + 热键）

**工作量估计**：2–4 周（含逆向），维护成本高。

**风险**：联机可能封号；仅建议本地/单机场景。

---

### 方案三：平台存档编辑器

**原理**：解析 KK 对战平台本地/服务器存档，修改 JSON/二进制字段后写回。

**本图适用性：❓ 未知（取决于存档位置与加密）**

| 优点 | 缺点 |
|------|------|
| 可改永久成长数据 | 存档格式未公开 |
| 不修改地图文件 | 可能有签名校验 |
| 用户体验好（改一次永久生效） | 可能存于云端，本地无文件 |

**前置条件**：
- 你需要提供：玩过该图后的本地存档路径，或平台缓存目录
- 确认哪些属性是存档驱动（进化等级？天赋？）

**工作量估计**：1–3 周（若格式可破解）；否则不可行。

---

### 方案四：脚本逆向 + 重打包（完整改图）

**原理**：获取完整 JASS/Lua 源码 → 修改数值表 → 重新编译进地图。

**本图适用性：⛔ 当前不可行**

| 障碍 | 说明 |
|------|------|
| `war3map.j` 仅 116 字节 | 只是加载器桩 |
| 无 `(listfile)` | 1500+ 资源文件名未知 |
| IMPLODE 压缩 | 需 StormLib / w3x2lni 等工具链 |
| 云脚本 | 核心逻辑可能根本不在本地 |

**若你拥有未加密原版或作者提供的脚本**，此方案才可重新评估。

**工作量估计**：4–8 周+，且可能最终失败。

---

### 方案五：基于魔兽 JAPI/Lua 插件的「本地启动器」

**原理**：本地 War3 + 注入 DLL（如 YDWE、Lua 插件）+ 自定义补丁脚本，在地图加载后 hook 原生 API。

**本图适用性：⚠️ 中等，仅限本地**

| 优点 | 缺点 |
|------|------|
| 可 hook `SetHeroStr` 等原生函数 | 平台版无法使用 |
| 不依赖解包完整脚本 | 需 C++/Lua 插件开发经验 |

**工作量估计**：3–6 周。

## 3. 推荐路径（按优先级）

在你回答 [03-待确认问题.md](./03-待确认问题.md) 之前，**仅作初步建议**：

```
若目标是「单机/本地练习，局内改属性」
  → 优先：方案二（内存辅助）
  → 备选：方案五（本地插件）

若目标是「改永久存档（进化等级、天赋等）」
  → 优先：方案三（存档编辑）
  → 需你提供存档样本

若目标是「做一张给别人玩的修改版地图」
  → 先确认「修改版」现状 + 是否能拿到完整脚本
  → 否则不可行

若目标是「改地图默认平衡（全体玩家）」
  → 本图不适合；需作者脚本或换一张本地完整脚本地图
```

## 6. 与仓库现有「神龙阁」项目的关系

当前 `war3tools` 仓库 `docs/` 目录规划的是 **Y3 编辑器新项目「神龙阁」**，与 **「小偷进化」** 是不同地图、不同技术栈：

| 项目 | 编辑器 | 状态 |
|------|--------|------|
| 神龙阁 | 网易 Y3 | 纯规划，未建工程 |
| 小偷进化 | 经典 WC3 + KK 平台 | 本次分析对象 |

两者可参考设计思路（如 30 分钟单局、进化机制），但**代码和工具不应混用**。本次所有产出已隔离在 `tools/xiaotou-jinhua/` 下。
