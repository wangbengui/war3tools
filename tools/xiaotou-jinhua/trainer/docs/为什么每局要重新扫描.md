# 为什么每局都要重新扫描？

## 简短回答

**不是。** 新一局游戏里，「攻击偷金」几乎一定在 **不同的内存地址**。

你截图里的 `0x2cb40290` 只属于 **那一局、那一刻**。换局后游戏会重新分配内存，地址会变（通常仍是 `0x2xxxxxxx` 这类堆地址，但具体数字不同）。

---

## 原理

| 概念 | 说明 |
|------|------|
| 堆内存 | 地图脚本每局为玩家属性 `new` 一块内存，局结束释放 |
| 动态地址 | `0x2cb40290` 不是固定表，每局随机/递增分配 |
| 进程不变 | `War3.exe` 没关，但 **地图逻辑重新初始化**，数据在新地址 |

类比：第一局属性在「临时房间 290 号」；第二局换到「临时房间 518 号」——同一属性，不同门牌号。

---

## 第二局搜不到常见原因

1. **还在用上一局的思路**  
   - 对旧地址写入 → 无效或崩溃  
   - 没点「首次扫描」，直接「再次扫描」→ 在上局结果里找，当然找不到  

2. **没清空扫描会话**  
   - 工具里还留着上局的匹配列表 → 点 **「新对局」** 再扫  

3. **数值或类型不对**  
   - 攻击偷金用 **float**  
   - 必须填 **本局当前** 数值（如 1850.32），不是上局的 2290.65  

4. **本局数值还没稳定**  
   - 刚开局 UI 数字在涨，等数值停住再扫  

5. **显示值 ≠ 内存值（较少见）**  
   - 若多次首次扫描都是 0 或上万条，可能存的是整数放大或别种编码，需再分析  

---

## 正确操作流程（每一局都要做）

```
1. 进入新对局
2. 修改器点「新对局」（清空上局结果）
3. 看游戏内当前「攻击偷金」→ 类型 float → 首次扫描
4. 等数值变化 → 填新值 → 再次扫描
5. 剩 1 条 → 双击修改
```

**每一局重复 3～5 步**，不能跳过首次扫描。

---

## 想「只扫一次、局局能用」怎么办？

需要 **指针扫描（Pointer Scan）**：找一条从 `Game.dll` 等固定模块出发的偏移链，指向属性地址。  
这是 Cheat Engine 进阶功能，和本工具当前「按数值搜地址」是两套方案。

若你希望，我可以在后续版本里加「保存指针链 / 一键定位攻击偷金」，但需要你在本机配合做一次指针扫描实验。
