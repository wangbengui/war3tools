# AI 自动化开发路径

> 目标：你极少参与，AI 完成绝大部分制作工作  
> 结论：**无法 100% 零参与**，但可压缩到 **一次性 1 小时搭建 + 之后偶尔点运行/上传**

---

## 为什么不能完全零参与？

Y3 地图不是纯代码项目。工程里包含：

- `header.project` — 工程入口（须编辑器创建）
- `maps/EntryMap/` — 关卡地形、物编、UI（大量 JSON/二进制，官方用编辑器或 **Y3 开发助手 MCP** 操作）
- KK 平台上传、开发者认证 — 必须真人登录

**仅靠 Cursor 在 `war3tools` 里写文件，无法生成可运行的 Y3 工程。**

---

## 官方已提供的 AI 自动化方案

网易为 Y3 做了 **Y3 开发助手**（VSCode 插件）+ **MCP 工具链**，支持：

| 工具层 | 能力 |
|--------|------|
| y3editor | 地形、物编、UI、资源、改 JSON（40+ 工具） |
| y3-helper | `launch_game`、`execute_lua`、`capture_screenshot` |
| y3runtime | UI 自动化测试 |

文档：
- AI 环境配置：https://163.com/y3/docs/guides/Editor/Y3AI
- Y3 开发助手：https://163.com/y3/docs/guides/FunctionManual/AIGC/Y3_AI

配合 **CodeMaker Agent**，可用自然语言驱动：建物编、画 UI、写 Lua、启动游戏、截图验 bug。

## 推荐分工：你 vs AI

### 你只需要做的一次性工作（约 1 小时）

1. 安装 **KK 客户端** → 内置 **Y3 编辑器**
2. 编辑器登录 → **新建项目**（选「生存」或「塔防」模板）→ 记住工程路径
3. 安装 **VSCode** + **Y3 开发助手** 插件 + **Git**
4. （可选）安装 **Python** 并加入 PATH
5. VSCode 打开工程父目录（含 `header.project` 的文件夹）
6. Y3 开发助手 → 选择 `header.project` → **初始化 Y3 库**
7. 在 Cursor 中配置 **Y3 MCP**（按开发助手文档连接）
8. 注册 https://create.kkdzpt.com/map 开发者（手机验证）

### 之后 AI 可做（你不必写代码）

- 创建/修改英雄、宠物、怪物物编
- 搭建 HUD、属性面板、商店 UI
- 编写 `script/` 下全部 Lua（偷金、波次、宠物拾取等）
- 编写 `cloud_script/` 云脚本（防作弊、结算）
- 启动游戏、截图、读日志、修 bug 循环
- 在 `war3tools` 维护设计文档与脚本，再同步到 Y3 工程

### 你仍需偶尔参与

| 频率 | 事项 |
|------|------|
| 每周 1–2 次 | 看 AI 截图/录屏，说「可以/不行」 |
| 上架前 | 编辑器内点「上传」、后台填宣传图与说明 |
| 出问题时 | 编辑器打不开、账号异常 — 需人工处理 |

## 两种协作模式

### 模式 A：Y3 开发助手 MCP（推荐，自动化最高）

```
你：新建空工程 + 配好 MCP（一次性）
    ↓
Cursor / CodeMaker：通过 MCP 改物编/UI/脚本 → 启动游戏 → 截图验证
    ↓
你：看图点头或提修改意见
```

### 模式 B：仅 war3tools + Cursor（无 Y3 环境）

```
AI 只能写：文档、Lua 源码、cloud_script 模板
    ↓
你必须：自己建工程、复制粘贴脚本、在编辑器里点保存运行
```

**模式 B 无法「全自动化」，只适合你会偶尔开编辑器。**

## war3tools 仓库的定位

建议作为 **「设计与脚本源」**，与 Y3 工程并列：

```
war3tools/                    Y3 工程（KK 目录下）
├── docs/          ──设计──→   （人工/AI 读 GDD）
├── script/        ──同步──→   maps/EntryMap/script/
└── cloud_script/  ──同步──→   cloud_script/
```

AI 在 `war3tools` 写好脚本后，复制或通过脚本同步到 Y3 工程；物编/UI 仍依赖 MCP 在 Y3 工程内直接改。

## 现实预期

| 目标 | 可行性 |
|------|--------|
| 你不写一行代码 | ✅ 可以（AI + Y3 助手） |
| 你不装任何软件 | ❌ 不行 |
| 你不新建工程 | ❌ 不行（至少点一次「新建」） |
| 你不注册开发者 | ❌ 不行（无法上架） |
| 你不测试 | ⚠️ 可以但质量无保障 |

**务实目标**：一次性 1 小时环境 + 每周 30 分钟验收反馈 ≈ **95% 工作由 AI 完成**。
