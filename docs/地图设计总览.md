# 地图设计总览

> KK 对战平台 · Y3 编辑器 · 魔兽 RPG 地图  
> 参考：小偷进化系列（无建造、英雄+助手、攻击偷金、波次 TD）  
> 个人记录与决策见 [更新文档.md](./更新文档.md)

---

## 1. 产品定位

**一句话**：攻击偷金 + 波次防守 + 三阶进化 + 助手辅助的动作 TD，通过存档与商城实现跨局成长。

| 项目 | 决策 |
|------|------|
| 编辑器 | Y3 |
| 开发方式 | Cursor + Windows + Y3 开发助手 |
| 人数 | 1–4 人（各人数均可通关） |
| 单局时长 | 约 30 分钟，支持提前挑战 BOSS |
| 首发难度 | N1 / N2 / N3 |
| 局内进化 | 3 阶 |
| 助手 | 纯辅助，无攻击，无敌 |
| 当前阶段 | 纯规划，尚未创建 Y3 工程 |

---

## 2. 核心循环

```
选难度 → 初始化英雄/助手/UI → 10秒倒计时
    → 波次刷怪（自动攻击 + 偷金）
    → 获得金币/杀敌/经验/掉落
    → 商店买 Buff / 技能 / 装备
    → 助手捡装 + 合成 + 光环
    → 每 5 波金币 BOSS → 挑战副本
    → 三阶进化（外观 + 数值跃升）
    → 第 30 波 / 提前 BOSS → 结算 → 存档写入
```

**核心爽点**：攻速偷金、进化视觉冲击、装备合成 Max、存档/商城带来的局外成长。

---

## 3. 设计原则

1. **主属性清晰**：力量驱动攻击与偷金，敏捷/智力提供概率触发。
2. **数值可读**：TAB 面板能看懂下一档提升；商店 Buff 以百分比为主。
3. **成长分层**：局内（等级+进化+装备）与局外（天赋+商城）分开叠加。
4. **敌人服务经济**：小怪 = 经验+掉落；金币 BOSS = 爆发经济；精英/BOSS = 门槛检验。
5. **MVP 先少后多**：1 英雄、5 类小怪变体、12 件装备线、8 个商店 Buff。

---

## 4. 开局流程

### 4.1 进入游戏（0–30 秒）

1. 选择难度 **N1 / N2 / N3**
2. 初始化英雄、助手、NPC 商店、HUD
3. 读取服务器存档、商城道具、天赋加成
4. 倒计时 **10 秒** → 第 1 波开始

### 4.2 前 5 分钟节奏

| 时间 | 玩家行为 | 系统事件 |
|------|----------|----------|
| 0–1 min | 熟悉操作、买首个 Buff | 第 1–2 波 |
| 1–3 min | 攒金币、买技能、助手捡装 | 第 3–4 波 |
| 3–5 min | 合成装备、考虑 1 阶进化 | 第 5 波 + 金币 BOSS |

## 5. 难度设计

| 难度 | 定位 | 怪物强度 | 存档收益 | 解锁 |
|------|------|----------|----------|------|
| N1 | 新手 | ×1.0 | 基础天赋点 | 默认 |
| N2 | 标准 | 生命×1.6 攻击×1.3 | 更多天赋点 | 通关 N1 |
| N3 | 硬核 | 生命×2.5 攻击×1.6 | 稀有天赋 | 通关 N2 |

后续扩展：N4+、无尽模式（N1–N3 稳定后）。
