# 平台与防作弊

> 上级文档：[地图设计总览](../地图设计总览.md)

Y3 + KK 平台接入：存档、云脚本、防刷、上架流程。

---

## 1. 防作弊策略

| 敏感逻辑 | 方案 |
|----------|------|
| 随机掉落 / 抽奖 | Y3 **服务器随机池** |
| 金币 BOSS / 通关奖励 | **云脚本**校验后发奖 |
| 天赋点 / 通关次数 | 存档防刷 + **仅云脚本可写** |
| 商城道具数量 | 平台托管，客户端只读 |
| 局内金币增量 | 结算时云脚本校验单局上限 |

---

## 2. 存档防刷配置清单

- [ ] 数值型槽位配置「仅允许增加」或云脚本写入
- [ ] 天赋点发放仅在云脚本 `_roomover` 事件
- [ ] 随机掉落走服务器随机池
- [ ] 开发期开启不同步检测日志

文档：[Y3 存档设置](https://163.com/y3/docs/guides/FunctionManual/SystemFunction/archive-setting)

---

## 3. 云脚本规划

工程目录：`cloud_script/`（与 Y3 地图工程同步）

| 职责 | 说明 |
|------|------|
| 房间结算 | 天赋点、通关次数写入 |
| 金币 BOSS | 伤害统计 → 金币奖励 |
| 敏感掉落 | 装备挑战结果校验 |
| 道具消耗 | `MsTryConsumeItem` |

文档：[Y3 云脚本](https://163.com/y3/docs/guides/FunctionManual/CloudScript)

---

## 4. 技术实现要点（Y3）

| 模块 | 实现 |
|------|------|
| 波次 | 全局计时器 + 刷怪配置表 |
| 偷金 | 攻击命中 → 按属性公式加金币 |
| 助手 | 跟随 + 周期扫描地面物品 |
| 合成 | 检测 3 同级同部位 → 生成高一级 |
| 人数缩放 | 开局读玩家数 → 全局系数 |
| 存档 | 开局读 / 结算写 |

---

## 5. 上架流程

1. 注册 [KK 开发者](https://create.kkdzpt.com/map)
2. 作者之家配置存档 Key（与 [存档与天赋](./存档与天赋.md) 一致）
3. Y3 编辑器上传地图
4. 填写作品信息、宣传物料
5. 测试服（`-maptest`）自测 → 申请审核 → 发布
6. 上架前 24h 预配商城商品

开发协作见 [开发/AI自动化开发路径.md](../开发/AI自动化开发路径.md)。
